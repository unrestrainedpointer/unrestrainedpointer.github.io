Failure Control in Multipath Route Tracing

  1. 路由发现建模：我们把从源s到目标地址d的多路径路由建模成$G = (V,E)$。该图G对于我们而言此时是未知的，我们要通过探测形成它。
  2. 我们从一个空图$\hat{G}=({s},\phi)$开始一步步开始构建，边缘集$\Omega$表示$\hat{G}$中仍需进一步拓展的节点，初始时$\Omega={s}$，终止条件是$\Omega=\phi, \hat{G}=G$
  3. 在边缘集中随机选一个节点v，枚举它的后继极点集$\sigma(v)={v’\in V:(v, v’)\in E}$，注意此时的$\sigma(v)$我们是不知道的，只是在逻辑上我们是要枚举其中的节点，我们实际得到的是它的子集$C_v\in \sigma(v)$。
    1. 对于发现的每一个后继$c\in C_v$，如果是新节点就加进$\hat{V}$，即$c\notin\hat{V}\Rightarrow\hat{V}\leftarrow\hat{V}\cup{c}$，同时如果不是目的地，则再把c加入$\Omega$，
    2. 如此进行知道$\Omega=\phi$
    3. 如何得到v的后继节点c？$\Rightarrow$per-node discovery操作：经过一系列的设置，让探测包经过v发送到v的下一跳。
    4. 我们设随机变量$X_{v,i}$表示$trace_i$上的节点v的下一跳的可能返回结果，并且假设它服从$\sigma(v)$上的均匀分布，且对于不同的i，$X_{v, i}$相互独立
  4. 成功和失败概率
    1. 失败概率$Pr(\hat{G})\not ={G}$：从源到目的至少有一条路径被探测算法丢失了
    2. 对于单个节点$v_i$，设其失败概率为$\beta_{K_i}$
      1. 唯一影响$\beta_{K_i}$的量是$v_i$后继节点的数量$K_i=|\sigma(v_i)|$
      2. 因此单个节点成功的概率为：$1-\beta_{K_i}$
      3. 由于各个节点的探测相互独立，因此全部节点成功的概率为：$\prod_i(1-\beta_{K_i})$
      4. 因此整个算法的失败概率为：$\beta_{all}=1-\prod_i(1-\beta_{K_i})$
    3. 接下来考虑单个节点$\beta_{K}$（省略了i）的求解：
      1. 对单个节点探测失败是指对于该节点发现了少于$K$个后继
      2. 设算法在某一个节点的$K$个后继节点中发现$k$个的概率，记为$t_K^k$
      3. 因此在某一个节点失败的概率：$\beta_K=\sum_{k=1}^{K-1}t_K^k, K>1$
      4. 对于某个节点而言，已经发现了$k$个节点之后再发现一个新节点的概率是：$q_k=\frac{K-k}{K}$，停留在第$k$个节点的概率为$p_k=1-q_k=\frac{k}{K}$
      5. 定义状态$(i, k)$表示在某一个节点处尝试了$i$次后发现了$k$个后继节点的概率
  5. 如何在已知某个节点的情况下探测其后继节点？
    1. MDA一跳一跳进行直到目的地，在每一跳，算法用相关的流标记每一个接口
    2. 假设一条$trace={s, n_1, n_2,…,d}$
    3. 对于$n_i\in trace$，MDA从s发出多次以该节点为目的的探测，每次探测改变流标识。每次发现的节点也用流来标记。

My Idea of yarrp6 MDA

  1. IPv4和IPv6的multipath routing算法有哪些区别

    1. IPv4 multipath routing：
      1. Equal-Cost Multi-Path Algorithm

        使用数据包头部标识中的一些字段形成hash，根据hash的结果来选下一跳的路由。

        如果hash函数的结果在后继路径上是均匀分布的，就称为Equal-Cost Multi-Path Algorithm，否则称为Non-Equal-Cost Multi-Path Algorithm

        Modulo-N Hash: 选取头部字段进行模n哈希，当添加或删除下一跳的时候，$\frac{N-1}{N}$个流会被影响

        Hash-Threshold: Hash-Threshold路由器首先通过对标识流的数据包报头字段执行散列来选择密钥。在哈希函数的输出空间中，为N个下一跳分配了唯一的区域。通过比较哈希值与区域边界，路由器可以确定哈希值属于哪个区域，从而确定使用哪个下一跳。这种方法的优点是，当添加或删除下一跳时，只影响区域边界(或阈值)附近的流。对于ECMP，哈希阈值的查找可以通过一个简单的除法(hash_value / fixed_region_size)来完成。当添加或删除下一跳时，所有流的1/4到1/2之间会改变路径。对该方法的分析可以在[2992]中找到。

        Highest Random Weight (HRW): 路由器通过对标识流的包头字段和下一跳地址执行哈希运算，为每一下一跳计算一个密钥。路由器选择结果键值最高的下一跳[4]。这样做的好处是可以最大限度地减少受下一跳添加或删除影响的流的数量(只有1/N)，但成本大约是模N散列的N倍。

        实现中通常采用五元组{dest addr, source addr, protocol, dest port, source port}
